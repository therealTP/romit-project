Wallet Directive Demo Page

Two separate versions of the wallet directive, using the same template and styling, but receiving wallet data in different ways. The result is the same, it just shows two solutions to the same task.

Each layout includes a placeholder div to represent the "welcome/profile completion" component. This will simulate how the wallet will look on the new dashboard.

Version 1

Wallet data passed in to directive scope from parent scope (the Demo state/ controller) via a dynamic/two-way scope binding (using "=") on the directive definition object.

Welcome/ profile completion component.

Version 2

Wallet data injected directly into directive scope from a wallet service via its controller, keeping its scope isolated.

Welcome/ profile completion component.
